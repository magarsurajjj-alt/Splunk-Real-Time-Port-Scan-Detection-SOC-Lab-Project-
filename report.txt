# 🔥 SECURITY INCIDENT REPORT  
## Splunk Port Scan Detection (Post-Incident Analysis)

---

# 📌 1. Incident Overview

A potential **port scanning activity** was detected in the internal network using Splunk SIEM. The activity originated from a Kali Linux machine targeting a Windows system.

The behavior is consistent with **reconnaissance activity using Nmap scanning tools**.

---

# 🧾 2. Incident Details

```text
Incident Type: Port Scan / Network Reconnaissance
Detection Tool: Splunk Enterprise
Severity: Medium
Status: Resolved (Lab Simulation)
Detection Time: Real-time
🧑‍💻 3. Affected Systems
Attacker Host: 192.168.1.71 (Kali Linux)
Target Host: 192.168.1.70 (Windows Machine)
Log Source: Windows Firewall Logs (pfirewall.log)
⚔️ 4. Attack Summary

The attacker performed a TCP/UDP port scan against the target system using Nmap.

Command used:
nmap -Pn -p 1-1000 192.168.1.70
Observed behavior:
Multiple ports probed in short time
Sequential scanning pattern
High number of connection attempts
Multiple destination ports accessed (135, 139, 445, 5432, 8000, 8089)
🔍 5. Detection Method

Splunk detected the activity using firewall log correlation and aggregation.

SPL Query Used:
index=firewall src_ip!="127.0.0.1"
| bucket _time span=1m
| stats dc(dest_port) as unique_ports 
        values(dest_port) as scanned_ports 
        count as attempts 
        by src_ip dest_ip _time
| where unique_ports >= 5
Detection Logic:
Groups traffic by source and destination IP
Counts unique destination ports
Flags behavior when threshold ≥ 5 ports within 1 minute
🚨 6. Alert Trigger Information
